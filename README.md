# Crypto Spread Bot

Pet project на Go для мониторинга спредов между криптобиржами и отправки уведомлений в Telegram.
Ключевая особенность проекта — поддержка поиска спредов между разными монетами, что полезно в случаях, когда одна и та же монета на разных биржах может иметь разные названия.

## Что делает проект

Система состоит из трёх сервисов:

- **bot** — Telegram-бот, который принимает команды пользователя, создаёт задачи мониторинга и умеет рассчитывать процент спреда между двумя ценами;
- **monitoring** — сервис, который подключается к биржам по WebSocket, собирает котировки и ищет спреды;
- **notification** — сервис, который получает события из Kafka, обрабатывает их пулом воркеров и отправляет уведомления в Telegram.

## Архитектура

Проект состоит из трёх основных сервисов:

### 1. Telegram-бот

Telegram-бот отвечает за взаимодействие с пользователем:
- принимает команды на создание задач мониторинга спреда;
- сохраняет созданные задачи в PostgreSQL;
- предоставляет отдельную команду для расчёта процента спреда между двумя ценами.

### 2. Monitoring-сервис

Monitoring-сервис отвечает за сбор рыночных данных и поиск спредов:
- загружает активные задачи из PostgreSQL;
- подписывается на потоки котировок бирж;
- сравнивает цены между разными биржами;
- при обнаружении подходящего спреда формирует событие и отправляет его в Kafka.

### 3. Notification-сервис

Notification-сервис отвечает за доставку уведомлений:
- читает события из Kafka;
- помещает события в Redis queue для дальнейшей обработки;
- обрабатывает очередь несколькими воркерами параллельно;
- автоматически масштабирует количество воркеров в зависимости от текущей нагрузки;
- отправляет уведомления пользователям в Telegram;
- ограничивает повторные уведомления по одной задаче в течение заданного интервала времени.

## Пример использования

### Создание задачи мониторинга

```text
Монитор PTB >0.5 <5
https://www.binance.com/ru/futures/PTBUSDT
https://www.gate.com/ru/futures/USDT/PTB_USDT
https://www.mexc.com/uk-UA/futures/PTB_USDT
```

После такой команды бот создаёт задачу на мониторинг спреда для монеты `PTB` между указанными биржами и будет отправлять уведомление, если спред окажется в диапазоне от `0.5%` до `5%`.

### Расчёт процента спреда

```text
% 0.2 0.3 500
```

Эта команда рассчитывает процент спреда между двумя ценами и потенциальную прибыль для указанного объёма позиции.

## Запуск через Docker Compose

Проект запускается в два этапа: сначала поднимается инфраструктура, затем сервисы приложения.

### 1. Запуск инфраструктуры

```bash
docker compose -f docker-compose.infra.yml up -d
```

Этот файл поднимает:
- PostgreSQL
- Redis
- Zookeeper
- Kafka
- Kafka UI

### 2. Запуск сервисов приложения

```bash
docker compose -f docker-compose.services.yml up --build -d
```

Этот файл запускает:
- `bot` — Telegram-бот
- `monitoring` — сервис мониторинга котировок
- `notification` — сервис обработки и отправки уведомлений

### Остановка

Остановить сервисы приложения:

```bash
docker compose -f docker-compose.services.yml down
```

Остановить инфраструктуру:

```bash
docker compose -f docker-compose.infra.yml down
```

## Переменные окружения

Пример файла конфигурации находится в `.env.example`.

Основные переменные:
- `DB_URL` — строка подключения к PostgreSQL;
- `REDIS_URL` — адрес Redis;
- `REDIS_PASSWORD` — пароль Redis, если используется;
- `KAFKA_BROKERS` — адрес Kafka broker;
- `KAFKA_TOPIC_NOTIFICATIONS` — топик для событий уведомлений;
- `MIN_WORKER` — минимальное количество воркеров в notification-сервисе;
- `MAX_WORKER` — максимальное количество воркеров в notification-сервисе;
- `TG_TOKEN` — токен Telegram-бота;
- `EXCHANGES` — список поддерживаемых бирж.
